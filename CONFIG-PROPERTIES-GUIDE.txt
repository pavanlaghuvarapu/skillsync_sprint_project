========================================================
RULE: WHAT GOES WHERE
========================================================

LOCAL application.properties (each service):
  - server.port
  - spring.application.name
  - spring.config.import=configserver:http://localhost:8888

GITHUB CONFIG REPO (skillsync-config):
  - Everything else:
    - Datasource/DB configs
    - JWT secret
    - Redis config
    - RabbitMQ config
    - Eureka client URL
    - Feign/Resilience4j config
    - Swagger config
    - file.upload-dir

DO NOT DUPLICATE: Never put DB/Redis/RabbitMQ in local properties!

review-service:       http://localhost:8086/swagger-ui.html
group-service:        http://localhost:8087/swagger-ui.html

HOW TO USE SWAGGER WITH JWT:
1. Open any service's swagger-ui.html
2. Click "Authorize" button (lock icon top right)
3. Enter: Bearer YOUR_JWT_TOKEN
4. Click Authorize
5. Now all APIs in that service work with your token!

GET TOKEN FLOW IN SWAGGER:
1. Open http://localhost:8081/swagger-ui.html (auth-service)
2. Find POST /api/auth/login
3. Click "Try it out"
4. Enter email and password
5. Click Execute
6. Copy "token" from response
7. Click Authorize -> paste token
8. Test all other endpoints!
